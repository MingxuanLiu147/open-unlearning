## 新增需求

### 需求：一键式遗忘学习流水线运行器
系统应提供一个 Python 流水线运行器，该运行器使用现有的 train.py 和 eval.py 入口点以及 Hydra 配置，编排遗忘学习流程，然后进行评估。

#### 场景：使用配置运行流水线
- **当** 用户使用有效的实验配置路径运行流水线时
- **那么** 系统首先执行遗忘学习，然后使用相同的配置模式进行评估，并在配置的保存目录下生成输出

#### 场景：缺少数据且不自动下载
- **当** 所需数据集缺失时
- **那么** 系统报告缺失的数据，在 v1 版本中不尝试自动下载

### 需求：TOFU 演示配置和灵活的数据分割
系统应包含两个默认的 TOFU 演示配置（Llama-3.2-1B Instruct 和 Qwen3-1.7B Base），并应允许通过配置或 CLI 覆盖来覆盖 TOFU 数据分割。

#### 场景：使用默认演示配置
- **当** 用户运行演示配置而不进行覆盖时
- **那么** 系统使用该配置中定义的默认模型和 TOFU 分割值

#### 场景：覆盖 TOFU 数据分割
- **当** 用户为 forget/retain/holdout 分割提供覆盖值时
- **那么** 流水线在数据集加载和评估中使用覆盖的分割值
