## 新增需求

### 需求：编辑工作流（单个、顺序、批量）
系统应支持单个编辑、顺序编辑和批量编辑的知识编辑工作流，适用于文本和多模态上下文。

#### 场景：单个编辑执行
- **当** 用户选择单个编辑任务和方法时
- **那么** 系统执行编辑并评估编辑结果

#### 场景：顺序编辑执行
- **当** 用户选择顺序编辑模式时
- **那么** 系统按顺序应用编辑并报告累积结果

### 需求：编辑数据集和方法覆盖
系统应包含 ZSRE、CounterFact、ELKEN、ConceptEdit 和长文本编辑（AKEW/UNKE/LEME）的数据集和方法钩子，并应允许通过注册表配置添加其他数据集和方法。

#### 场景：添加新的编辑数据集
- **当** 用户添加新的编辑数据集配置并注册它时
- **那么** 系统可以在不修改核心代码的情况下使用该数据集执行编辑任务

### 需求：编辑指标
系统应计算并报告编辑指标，包括可靠性、局部性、泛化性和可移植性（包括多模态变体）。

#### 场景：报告编辑指标
- **当** 编辑评估完成时
- **那么** 系统为每个任务报告所需的编辑指标
