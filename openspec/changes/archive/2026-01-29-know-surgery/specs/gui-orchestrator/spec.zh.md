## 新增需求

### 需求：GUI 在 v2 结束时启动并共享配置模式
系统应在 v2 结束时实现基于 Gradio 的 GUI，并应使用与 CLI 相同的配置模式，以实现导入/导出兼容性。

#### 场景：导入 CLI 配置
- **当** 用户在 GUI 中上传 CLI 配置文件时
- **那么** GUI 加载配置并准备可运行的设置

### 需求：UltraRAG 风格的向导流程（渐进式）
GUI 应提供向导式流程，用于选择模型、方法、数据集和评估，并应在配置完成后允许一键运行。

#### 场景：基于向导的运行
- **当** 用户完成向导选择时
- **那么** GUI 启动流水线运行并显示进度

### 需求：结果可视化（渐进式）
GUI 应提供运行状态和关键指标的基本可视化，并随时间扩展。

#### 场景：显示运行摘要
- **当** 流水线运行完成时
- **那么** GUI 显示指标摘要和指向输出产物的链接
