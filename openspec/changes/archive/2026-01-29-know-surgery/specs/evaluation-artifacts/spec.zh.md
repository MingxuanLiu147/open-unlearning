## 新增需求

### 需求：标准化评估产物
系统应为每次运行生成标准化的评估产物，包括清单、聚合指标和可选的样本追踪。

#### 场景：生成清单
- **当** 运行完成评估时
- **那么** 系统写入一个清单，包含配置快照、模型标识符、随机种子和运行时环境信息

#### 场景：生成指标
- **当** 评估完成时
- **那么** 系统写入一个指标文件，包含每个指标的结果和聚合分数

### 需求：缺少保留日志时的优雅降级
系统应允许在保留日志缺失时继续评估，并应清楚地标记缺失的依赖指标。

#### 场景：缺少保留日志
- **当** 未为依赖保留日志的指标提供保留日志时
- **那么** 系统跳过这些指标，并在输出中将其记录为缺失
